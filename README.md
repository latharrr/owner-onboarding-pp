# Picapool PG Owner Onboarding System

A production-ready, mobile-first onboarding tool for Picapool field interns. Onboard PG owners in under 2 minutes with auto-save, offline support, and Google Sheets integration via Apps Script.

---

## Tech Stack

- **Next.js 15** — App Router, TypeScript, Turbopack
- **Tailwind CSS v4** — CSS-first configuration
- **shadcn/ui** — Nova preset with Radix primitives
- **Zustand** — Global state with localStorage persistence
- **React Hook Form + Zod** — Form validation
- **Framer Motion** — Minimal page transitions
- **Google Apps Script** — Sheets backend (duplicate check, ID generation, writes)
- **Vercel** — Deployment + Analytics
- **idb** — IndexedDB offline queue

---

## Setup Guide

### 1. Google Sheets

1. Create a new Google Spreadsheet at [sheets.google.com](https://sheets.google.com)
2. Copy the Spreadsheet ID from the URL:
   ```
   https://docs.google.com/spreadsheets/d/SPREADSHEET_ID_HERE/edit
   ```

### 2. Google Apps Script

1. Go to [script.google.com](https://script.google.com) → **New Project**
2. Delete the default `myFunction()` code
3. Open `apps-script/Code.gs` from this repo and paste its contents
4. Replace the constants at the top:
   ```javascript
   const SPREADSHEET_ID = 'your-spreadsheet-id';
   const SHARED_SECRET = 'your-long-random-secret';
   ```
5. **Deploy** → **New deployment** → **Web App**
   - Execute as: **Me**
   - Who has access: **Anyone**
6. Click **Deploy** and copy the **Web App URL**

### 3. Environment Variables

```bash
cp .env.example .env.local
```

Fill in `.env.local`:
```env
APPS_SCRIPT_WEB_APP_URL=https://script.google.com/macros/s/YOUR_ID/exec
APPS_SCRIPT_SECRET=your_shared_secret_here
NEXT_PUBLIC_APP_URL=https://your-app.vercel.app
```

### 4. Install & Run

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

### 5. Deploy to Vercel

1. Push to GitHub
2. Import project in [vercel.com](https://vercel.com)
3. Add environment variables in Vercel Dashboard → Settings → Environment Variables
4. Deploy!

## Google Sheets Structure

The Apps Script auto-creates 4 tabs:

| Tab | Description |
|---|---|
| `Owners` | One row per owner — name, phone, GPS, visit status |
| `Properties` | All property details including amenities, food, rules |
| `RoomConfigurations` | Room types with rent pricing |
| `Submissions` | One row per complete submission |

### ID Format

IDs are generated by Apps Script (never client-side):
- Owners: `OWN-001`, `OWN-002`...
- Properties: `PRP-001`, `PRP-002`...
- Submissions: `SUB-001`, `SUB-002`...

UUIDs are also stored for internal linking.

## Onboarding Flow

```
Welcome
  ↓
Owner Details (+ duplicate phone check)
  ↓
Owner Summary (edit + add properties)
  ↓
Property: Identity → Audience → Capacity → Room Configs →
          Amenities → Food → Rules → Financials →
          Availability → Internal Notes
  ↓
Add Another Property? (YES → back to property steps)
  ↓
Review Missing Information (✓/⚠ checklist)
  ↓
Submit → Success (with IDs + duration)
```

## Offline Support

- **Zustand + localStorage**: All form data persists through page reloads
- **IndexedDB queue** (`idb`): Failed API calls are queued and retried when online
- **Auto-detection**: Online/offline status is monitored via browser events

## AI PG Discovery

A second, independent feature reachable from the home page (**"Find a PG with AI"** → `/discover`) for prospective tenants — separate from the intern onboarding flow above. It lets anyone search the same Owners/Properties/RoomConfigurations sheet using natural language instead of manual filters (e.g. *"Girls PG under ₹12,000 near North Campus"*, *"Cheapest PG in Vijay Nagar"*, *"Compare Sunrise PG vs Green Valley PG"*).

**How it works — no vector DB, no embeddings:**

```
Google Sheet (Apps Script GET_ALL_DATA)
  → joined into Owner → Property → RoomConfig objects (src/lib/discover/data.ts, 2-min in-memory cache)
  → query parsed into structured filters with plain regex/keyword rules (src/lib/discover/filters.ts)
  → filtered/ranked in JS — only the matches are ever sent onward
  → compact, privacy-minimized JSON (no phone/email/address) sent to Groq with a fixed system prompt (src/lib/discover/groq.ts)
  → streamed answer + matched property cards rendered on /discover
```

The system prompt instructs the model to answer **only** from the supplied data and to say so explicitly when something isn't available — it never sees the full spreadsheet or raw sheet rows, only the already-filtered slice for the current query.

**Setup:**
1. Re-deploy `apps-script/Code.gs` (adds the read-only `GET_ALL_DATA` action) — same deployment steps as above.
2. Get a free key at [console.groq.com](https://console.groq.com) and add it to `.env.local`:
   ```env
   GROQ_API_KEY=your_groq_api_key_here
   ```
3. Visit `/discover`.

**Pages:** `/discover` (search), `/discover/property/[id]` (details + AI summary), `/discover/compare?ids=PRP-001,PRP-002` (side-by-side compare).
